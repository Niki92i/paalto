# Team Context

These files are deliberately plain markdown. Fill them with your company's product voice, design system, strategy, and engineering standards before a live workflow. Claude Code can read them as local context when running paalto agents.

Recommended order:

1. `product-voice.md`
2. `company-strategy.md`
3. `design-system.md`
4. `engineering-standards.md`

Keep secrets, customer PII, and private credentials out of these files unless the repo is private and your team explicitly accepts that risk.
